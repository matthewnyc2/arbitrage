What you're looking at

A categorical Polymarket event is a market like "Who wins the 2028 Election?" where every outcome is a separate YES token and exactly one outcome will win. In fair pricing, the sum of the cheapest offers across every outcome must equal $1.00 — because if you buy one share of each, the winning outcome pays you $1. When that sum dips below $1 for a moment, you can lock in the gap as risk-free profit. This page watches every active negRisk event's live order book and shows the math in real time.

opportunities this session

how this works

Your browser opens a WebSocket to Polymarket's public CLOB (ws-subscriptions-clob.polymarket.com) and subscribes to every outcome token of the events on the left. For each tick, it updates the local L2 book and recomputes the basket cost. No server, no backend, no keys — the page is the bot.

safety

This demo is read-only. It does not hold funds, sign, or submit orders. The production Python version in the repo gates order submission behind ARB_MODE=live plus an operator-toggled dry_run flag.
